# 闲置微铺（Idle）

> 一个轻量级的本地 / 校园二手闲置交易小平台：买家发布闲置、他人担保购买、平台托管资金、后台确认发货后结算。前后端一体，原生 HTML/CSS/JS + PHP + MySQL，零构建步骤，开箱即用。

---

## 一、项目简介

闲置微铺（代号 `xzwp`，英文名 **Idle**）是一个面向小范围（校园 / 社区 / 兴趣小组）的 C2C 二手交易平台，核心特色是**担保交易**：

- 买家下单后，款项先由「平台」托管（冻结在买家账户，不立即打给卖家）；
- 卖家在后台「订单管理」点击「确认发货」后，托管资金才结算给卖家；
- 若订单被「驳回」，款项原路退回买家，商品自动重新上架。

平台不碰真实资金通道，所有余额均在系统内部记账，适合作为**演示 / 学习 / 内部小范围使用**。

## 三、技术栈

- **前端**：原生 HTML / CSS / JavaScript + [Vue 3](https://v3.vuejs.org/)（Options API，本地引入 `vue.global.prod.js`，无打包构建）
- **后端**：PHP（PDO + MySQL），REST 风格接口（`api/*.php`）
- **后台**：独立的 `administerint/` 目录，与前端双会话隔离（前端 `uid` / 后台 `admin_id`）
- **存储**：MySQL（建表脚本 `api/install.sql`）
- **运行**：任意支持 PHP + MySQL 的 Web 服务器（Nginx / Apache）

## 五、安装与部署

1. **准备环境**：PHP 8.x + MySQL 5.7+，并启用 `pdo_mysql` 扩展。
2. **导入数据库**：在 MySQL 中创建库（如 `xzwp`），执行 `api/install.sql` 建表；或部署后访问 `api/setup.php` 自动初始化。
3. **配置数据库**：编辑 `api/config.php`，填写数据库主机、库名、用户名、密码。
4. **配置上传目录**：确保 `api/uploads/` 可写（Web 服务器用户有写权限）。
5. **部署站点**：将整个目录放到 Web 根目录（如宝塔建站指向 `xzwp/`），确保 `api/` 可通过 HTTP 访问。
6. **访问**：
   - 前端首页：`http(s)://你的域名/index.html`
   - 后台入口：`http(s)://你的域名/administerint/admin.html`

> 默认后台账号请在部署后于 `users` 表手动插入管理员，或通过后台注册首个账号后于数据库将其标记为管理员（具体字段见 `install.sql`）。

## 六、核心业务流程（担保交易）

### 6.1 发布保证金机制

发布商品时，系统会**从发布人余额冻结该商品的 `price`（发布保证金）**，仅受发布人自己的上下架动作控制：

- **上架（on）**：冻结 `price`，发布人余额相应减少；
- **主动下架（off）**：退回 `price`，发布人余额恢复；
- **重新上架（on）**：再次冻结 `price`；
- **被购买（sold）**：商品状态置为 `sold`，此时保证金**不再随上下架变动**，等待后台统一结算；
- **删除在售商品**：若是 `on` 状态会先退回保证金再删除。

> 这笔发布保证金与买家的付款、后台发货/驳回互不影响，直到后台「确认发货」时才连同订单金额一起结算给发布人。

### 6.2 担保交易闭环

```
买家浏览商品（发布人已上架并冻结发布保证金）
   │
   ▼
点击「购买」→ 弹窗确认
   │
   ▼
后端下单：扣减买家余额，订单状态 = pending（平台托管），商品状态置为 sold
   │
   ▼
后台「订单管理」：
   ├── 确认发货（ship）  → 管理员输入【实际打款金额】，打款给发布人，订单 completed（不可撤销）
   └── 驳回（reject）    → 退款退回买家余额，商品重新上架（on），订单 rejected
   │
   ▼
买家「我的订单」可见：待发货 / 已完成（含实际打款金额）/ 已驳回
```

**资金安全要点**

- 下单时**只扣买家、不立即打款**，资金处于「平台托管」状态，规避跑单风险。
- **发布保证金**仅在发布人上下架时冻结/退回，与担保交易资金流隔离，避免重复占用或错扣。
- 后台「确认发货」由管理员**手动输入实际打款金额**（默认 = 订单金额 + 押金），打款后订单不可逆结束，订单上展示「实际打款金额」。
- 充值采用**审批制**：用户提交充值后仅生成 `pending` 记录，需后台在「财务审核」中确认入账，余额才会增加。
- 提现为**申请制**：用户提交提现后冻结对应余额，需后台审批通过后实际出金（本项目为内部记账演示，未对接真实支付通道）。

## 七、前端使用说明

| 页面 | 操作 |
| --- | --- |
| 首页 | 浏览商品、轮播图展示（后台配置）、点击图片可放大保存、点「购买」进入担保流程 |
| 登录 / 注册 | 图形验证码校验后注册或登录；标题/图标由系统配置统一控制 |
| 个人中心 | 查看余额与头像（每次打开实时拉取服务端）、提现、联系客服 |
| 商品管理 | 发布闲置（含**押金**字段，上架即冻结对应余额、下架退回）、图片 120×120 正方形、编辑、下架/删除；已售商品锁定不可上下架 |
| 我的订单 | 按「全部 / 待发货 / 已完成 / 已驳回」筛选查看；已完成订单展示「已打款 ¥实际金额」 |
| 充值 / 提现 | 提交充值 / 提现申请，等待后台审核 |
