前端使用手册

面向普通用户（买家 / 卖家）。入口页面位于 page/ 目录，首页为站点根 index.html。

一、项目简介与整体流程

闲置微铺是一个 C2C 二手交易平台：用户注册登录后，可发布闲置商品、浏览购买、管理订单，并通过余额进行担保交易。资金由平台托管，买家付款后款项暂不打给卖家，待后台「确认发货」才结算，或「驳回」则退款。

1 注册/登录auth.php 鉴权
2 发布商品products.php 上架
3 浏览/购买order.php 扣款托管
4 后台发货admin 确认/驳回
5 结算打款 or 退款

三、核心操作流程

① 注册 / 登录

  1. 打开 page/login.html，点「注册」填写账号、密码、昵称（可选上传头像）。
  2. 注册/登录成功后进入个人中心，顶部显示头像、昵称与余额。

② 发布商品（卖家）

  1. 进入「管理」页 → 填写标题、价格、押金、描述，上传商品图（建议正方形 120×120 以上）。
  2. 提交后商品上架，系统立即从你的余额冻结该商品价格（发布保证金）；此时账户余额会减少对应金额。
  3. 下架会退回该笔保证金（余额恢复）；重新上架会再次冻结；已售出的商品会被锁定，不可再上下架，需等待后台结算。
发布保证金：上架即冻结商品价格，下架即退回，重新上架再冻结。这笔保证金与买家的付款、后台发货互不影响，最终在后台「确认发货」时连同订单金额一起打给你。

③ 购买（担保交易，买家）

  1. 首页点商品「购买」→ 弹窗确认 → 点「确认支付」。
  2. 系统立即扣减买家余额，资金进入平台托管（订单状态：待发货），同时商品自动下架。
  3. 在「订单管理」可查看订单状态。

④ 订单管理

  1. 「订单管理」按 Tab 切换：全部 / 待发货 / 已完成 / 已驳回。
  2. 待发货 = 平台托管中；已完成 = 后台已确认发货并打款给卖家，卡片上展示「已打款 ¥实际金额」；已驳回 = 后台驳回，款项已退回你的余额。

⑤ 充值 / 提现

  1. 个人中心点「充值」→ 填金额 → 提交后生成待审核充值单，后台审批通过余额才增加。
  2. 点「提现」→ 填金额 → 提交后冻结对应余额，生成待审核提现单，后台审批通过才实际打款。

⑥ 首页轮播 & 联系客服

  • 首页轮播图：由后台管理员在「轮播图」模块配置。有图片时显示图片轮播（可点击跳转链接）；未配置时显示默认渐变色占位。
  • 联系客服：个人中心显示「联系客服」入口（后台「系统配置」中填了联系方式才会出现），点击可复制客服微信/QQ/邮箱等。
  • 站点标题与图标：全站统一由后台「系统配置」控制，包括浏览器标签页标题、标签页小图标(favicon)、导航栏品牌名、登录页 Logo 等。
注意：充值与提现均为审核制，提交后不会立即到账，需等待后台管理员审批。余额不足时购买/提现会被拒绝。
小技巧：首页与订单里的图片均可点击放大，放大后可保存原图。

四、常见问题

  • 头像不显示？ 个人中心每次打开会实时从服务端拉取最新资料（含头像），若仍不显示请硬刷新（Ctrl+Shift+R）。
  • 购买没反应？ 确认已登录；未登录会提示先登录。
  • 钱扣了但订单还是待发货？ 正常，说明资金在平台托管，等待卖家发货、后台确认。
  • 提现多久到账？ 取决于后台审批速度，请留意「订单管理 / 个人中心」状态变化。
  • 首页轮播图不显示？ 需要后台管理员在「轮播图」中上传并启用。未配置时首页会显示默认渐变色占位。
  • 改了系统配置但没变化？ 配置保存在数据库，已打开的页面需刷新一次才能获取最新值；新打开的页面自动使用最新配置。
  • 浏览器标签页还是旧图标？ 系统配置保存后，favicon 会动态更新，但部分浏览器有强缓存，可 Ctrl+Shift+R 强刷或清缓存。
  • 发布后余额变少了？ 正常，发布商品时会冻结商品价格作为「发布保证金」，下架即退回，重新上架再冻结；最终在后台确认发货时连同订单金额一起打给你。

后端管理手册

面向运营 / 客服 / 财务。后台位于 administerint/ 目录，与前端使用独立的 admin 会话。

一、后台入口与登录

  1. 访问 administerint/admin.html（或各子页面，如 dashboard.html）。
  2. 使用管理员账号登录（与普通用户账号体系分离）。

三、担保交易处理流程（重点）

买家付款后，订单为 待发货（pending），款项在平台托管。后台需人工介入：

订单 pending买家已付款托管
确认发货打款给卖家
状态 completed交易完成

✅ 确认发货（打款）

  1. 进入「订单管理」，找到 待发货 订单。
  2. 点「确认发货」→ 在弹窗中输入实际打款金额（默认预填为「订单金额 + 押金」，可按实际交易调整）。
  3. 点「确认打款」→ 系统将输入的金额打入发布人（卖家）余额，订单变为 已完成 并锁定（不可撤销），订单列表「实际打款」列显示该金额。

⛔ 驳回（退款）

  1. 进入「订单管理」，点「驳回」→ 确认。
  2. 系统将托管款项退回买家余额，商品重新上架，订单变为 已驳回。
财务安全：「确认发货」会真实转移资金且操作不可撤销，请核实交易真实性后再输入金额打款；驳回会退款并恢复商品上架（状态 on，发布保证金仍冻结，卖家下架后可退回）。

四、充值 / 提现审核

前端提交的充值、提现均为待审核（financial_orders 表，status=pending）。

  1. 进入「财务管理」→ 切换「充值审核 / 提现审核」。
  2. 核对金额与账户信息，点「通过」或「驳回」。
  3. 充值通过：用户余额增加；驳回：不产生变动。
  4. 提现通过：此前已冻结的余额正式扣除；驳回：冻结的余额退回用户。
注意：提现提交时余额已被冻结（扣除）。若驳回，务必退回，否则用户资产受损。

五、用户与商品管理

  • 用户管理：可查看用户列表、封禁违规账号、必要时手动调整余额（用于补偿/纠错）。
  • 商品管理：审核发布内容，下架违规商品，删除垃圾信息。注意：已售（sold）商品由交易锁定，不可在后台直接上下架，需走订单发货/驳回流程。
  • 收款账户：在「收款账户」中维护平台的充值收款码 / 提现账户，前端充值页会读取展示。

六、轮播图管理

首页顶部的轮播图由后台统一管理，前端通过 API 自动获取展示。

  1. 进入「轮播图」→ 点「添加轮播图」。
  2. 上传图片（建议比例 3:1 或 2:1，支持 JPG/PNG，≤3MB）、填写标题（显示在图片底部蒙层上，选填）、跳转链接（点击后跳转的地址，选填）、排序（数字越小越靠前）。
  3. 保存后立即在前端首页生效。可随时启用/禁用单张轮播图（不删除数据），或直接删除。
提示：未配置任何轮播图时，前端首页会显示默认渐变色占位轮播，不会空白。

七、系统配置

全站品牌形象和联系方式在此统一设置，保存后所有页面即时同步更新。

📛 站点标题

  1. 填写站点名称（如「闲置微铺」）。
  2. 自动应用到：浏览器标签页标题、前端导航栏品牌名、前端/后台登录页大标题、后台侧边栏品牌名。

🖼 站点图标

  1. 上传正方形图标（PNG/SVG/JPG，≤2MB）。
  2. 自动应用为：浏览器 favicon（标签页小图标）、登录页 logo、导航栏品牌标识（有图显示图片，无图显示首字）。

📞 客服联系方式

  1. 填写客服渠道（如微信/QQ群/邮箱等）。
  2. 前端「个人中心」会显示「联系客服」入口，点击可复制联系方式。
  3. 留空则前端隐藏该入口。
注意：系统配置保存在数据库单行记录中（settings 表），修改后无需重启或刷新缓存，所有已打开的页面下次加载时即获取最新值。

八、日常运营 SOP（建议）

  1. 每日查看「仪表盘」掌握平台整体数据。
  2. 及时处理「订单管理」中的待发货订单（确认/驳回）。
  3. 及时审批「财务管理」中的充值 / 提现申请。
  4. 定期巡查「商品管理」「用户管理」，处置违规内容/账号。
  5. 根据运营需要更新「轮播图」（如活动推广、新品上架公告）。
  6. 站点更名或更换品牌形象时，通过「系统配置」一键更新全站标题和图标。
