前端使用手册

面向普通用户（买家 / 卖家）。入口页面位于 page/ 目录，首页为站点根 index.html。

一、项目简介与整体流程

闲置微铺是一个 C2C 二手交易平台：用户注册登录后，可发布闲置商品、浏览购买、管理订单，并通过余额进行担保交易。资金由平台托管，买家付款后款项暂不打给卖家，待后台「确认发货」才结算，或「驳回」则退款。

1 注册/登录auth.php 鉴权
2 发布商品products.php 上架
3 浏览/购买order.php 扣款托管
4 后台发货admin 确认/驳回
5 结算打款 or 退款

三、核心操作流程

① 注册 / 登录

  1. 打开 page/login.html，点「注册」填写账号、密码、昵称（可选上传头像）。
  2. 注册/登录成功后进入个人中心，顶部显示头像、昵称与余额。

② 发布商品（卖家）

  1. 进入「管理」页 → 填写标题、价格、描述，上传商品图（建议正方形 120×120 以上）。
  2. 提交后商品上架，首页即可被其他用户看到。

③ 购买（担保交易，买家）

  1. 首页点商品「购买」→ 弹窗确认 → 点「确认支付」。
  2. 系统立即扣减买家余额，资金进入平台托管（订单状态：待发货），同时商品自动下架。
  3. 在「订单管理」可查看订单状态。

④ 订单管理

  1. 「订单管理」按 Tab 切换：全部 / 待发货 / 已完成 / 已驳回。
  2. 待发货 = 平台托管中；已完成 = 后台已确认发货并打款给卖家；已驳回 = 后台驳回，款项已退回你的余额。

⑤ 充值 / 提现

  1. 个人中心点「充值」→ 填金额 → 提交后生成待审核充值单，后台审批通过余额才增加。
  2. 点「提现」→ 填金额 → 提交后冻结对应余额，生成待审核提现单，后台审批通过才实际打款。
注意：充值与提现均为审核制，提交后不会立即到账，需等待后台管理员审批。余额不足时购买/提现会被拒绝。
小技巧：首页与订单里的图片均可点击放大，放大后可保存原图。

四、常见问题

  • 头像不显示？ 个人中心每次打开会实时从服务端拉取最新资料（含头像），若仍不显示请硬刷新（Ctrl+Shift+R）。
  • 购买没反应？ 确认已登录；未登录会提示先登录。
  • 钱扣了但订单还是待发货？ 正常，说明资金在平台托管，等待卖家发货、后台确认。
  • 提现多久到账？ 取决于后台审批速度，请留意「订单管理 / 个人中心」状态变化。

后端管理手册

面向运营 / 客服 / 财务。后台位于 administerint/ 目录，与前端使用独立的 admin 会话。

一、后台入口与登录

  1. 访问 administerint/admin.html（或各子页面，如 dashboard.html）。
  2. 使用管理员账号登录（与普通用户账号体系分离）。

二、功能模块总览

模块页面职责
仪表盘dashboard.html平台总览数据（用户数、商品数、营业额、充值/提现统计）
商品管理products.html查看/审核/下架/删除全平台商品
订单管理orders.html担保交易核心：确认发货（打款）/ 驳回（退款）

财务管理finance.html充值单 / 提现单审核（通过/驳回）

三、担保交易处理流程（重点）

买家付款后，订单为 待发货（pending），款项在平台托管。后台需人工介入：

订单 pending买家已付款托管
确认发货打款给卖家
状态 completed交易完成

✅ 确认发货（打款）

  1. 进入「订单管理」，找到 待发货 订单。
  2. 点「确认发货」→ 确认。
  3. 系统将托管的款项打入卖家余额，订单变为 已完成。

⛔ 驳回（退款）

  1. 进入「订单管理」，点「驳回」→ 确认。
  2. 系统将托管款项退回买家余额，商品重新上架，订单变为 已驳回。
财务安全：「确认发货」会真实转移资金，请核实物流/交易真实性后再操作；驳回会退款并恢复商品上架。

四、充值 / 提现审核

前端提交的充值、提现均为待审核（financial_orders 表，status=pending）。

  1. 进入「财务管理」→ 切换「充值审核 / 提现审核」。
  2. 核对金额与账户信息，点「通过」或「驳回」。
  3. 充值通过：用户余额增加；驳回：不产生变动。
  4. 提现通过：此前已冻结的余额正式扣除；驳回：冻结的余额退回用户。
注意：提现提交时余额已被冻结（扣除）。若驳回，务必退回，否则用户资产受损。

五、用户与商品管理

  • 用户管理：可查看用户列表、封禁违规账号、必要时手动调整余额（用于补偿/纠错）。
  • 商品管理：审核发布内容，下架违规商品，删除垃圾信息。
  • 收款账户：在「收款账户」中维护平台的充值收款码 / 提现账户，前端充值页会读取展示。

六、日常运营 SOP（建议）

  1. 每日查看「仪表盘」掌握平台整体数据。
  2. 及时处理「订单管理」中的待发货订单（确认/驳回）。
  3. 及时审批「财务管理」中的充值 / 提现申请。
  4. 定期巡查「商品管理」「用户管理」，处置违规内容/账号。
